探讨命令行sqlplus无法登陆的问题

By welley

Sep 17 2014 更新日期:Sep 17 2014

文章目錄
  1. 1. 用户名/密码错误
  2. 2. 监听问题
  3. 3. 数据库服务

  5. 5. 数据库实例问题

命令行 sqlplus 无法登陆，常常是用户名/密码错误、监听配置错误或未启动、数据库服务名丢失等等原因

用户名/密码错误

找到自己设的密码

这全靠自己创建数据库实例时，备份或记住相关信息

若最后没找到，则删除该实例，重新配置一个数据库实例

注意，是数据库实例，而非整个 Oracle

监听问题

监听服务，监听指定网址和指定端口的 Oracle 服务

若是此类问题，则重新配置监听

重新配置，使用 Oracle 安装组件之一： Net Configuration Assistant

配置时，注意监听名称和端口，要与数据库实例一致

数据库服务

找不到指定服务，要使用 Oracle 安装组件之一重新配置： Net Configuration Assistant

注意，监听的网址、端口和服务名

环境变量问题

一般情况下，一个系统安装一个 Oracle

则系统变量 path 中，只有一个 Oracle 的相关路径

但安装两个以上的 Oracle， 则注意要设成指定数据库的相关路径

否则，会报 protocol adapter errror

数据库实例问题

一般情况下，是因为安装 Oracle 失败 或 配置 Oracle 实例造成错误

1、前者，重装 Oracle ，因为排查可能花很多时间（除非能马上找到问题原因，且能修复）

重装前提：断网，关防火墙，关杀毒软件

另个人见解：关其他软件，且不做其他操作，去休息或看实体书，别碰电脑

2、后者，重新配置 Oracle 实例

注意，备份设定的密码，字符集，用途等

errorfaillistenerloginservicesqlplus
